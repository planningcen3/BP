# PEA Budget Control

เว็บลงคุมงบประมาณสำหรับบันทึกรายการเบิกจ่ายของแผนก พร้อมสรุปยอดคงเหลือ ยอดใช้ไป รายการล่าสุด และเชื่อมต่อ Google Sheet ผ่าน Google Apps Script

## ความสามารถหลัก

- เลือกชุดงบประมาณได้ระหว่างงบทำการและงบ คพจ.
- บันทึกรายการเบิกจ่ายตามหมวดงบ ชื่อ วันที่ รายละเอียด และยอดเงิน
- แสดง popup ตรวจสอบรายละเอียดก่อนยืนยันบันทึก เพื่อช่วยลดการกรอกผิด
- สรุปยอดงบประมาณ ยอดเบิกแล้ว ยอดคงเหลือ และเปอร์เซ็นต์การใช้
- แสดงสถานะรายหมวดงบและประวัติรายการเบิกล่าสุด
- ค้นหาหมวดงบและรายการเบิกได้
- ส่งออกรายการเป็น CSV
- บันทึกข้อมูลในเครื่องด้วย `localStorage`
- รองรับการส่งข้อมูลขึ้น Google Sheet ผ่าน Apps Script

## ไฟล์สำคัญ

- `index.html` โครงสร้างหน้าเว็บ
- `styles.css` รูปแบบหน้าตาและ responsive layout
- `app.js` logic การคำนวณงบ บันทึกรายการ popup ยืนยัน และ Google Sheet sync
- `assets/pea-mark.svg` โลโก้ PEA
- `google-apps-script/` โค้ดฝั่ง Google Apps Script สำหรับรับข้อมูลเข้า Google Sheet
- `DEPLOY.md` คู่มือ deploy เพิ่มเติม

## วิธีเปิดใช้งานในเครื่อง

โปรเจกต์นี้เป็น static website ไม่ต้องติดตั้ง package เพิ่ม

เปิดไฟล์ `index.html` ด้วย browser ได้ทันที หรือวางทั้งโฟลเดอร์บน static hosting เช่น GitHub Pages, Netlify หรือ Cloudflare Pages

## การบันทึกรายการเบิก

1. เลือกงบประมาณที่ต้องการใช้งาน
2. เลือกหมวดงบ
3. กรอกชื่อ วันที่ ยอดเบิก และรายละเอียดเพิ่มเติม
4. กด `บันทึกรายการเบิก`
5. ตรวจสอบ popup สรุปรายละเอียด
6. กด `ยืนยัน` เพื่อบันทึก หรือกด `แก้ไข` เพื่อกลับไปแก้ข้อมูลโดยยังไม่บันทึก

## การเชื่อมต่อ Google Sheet

โฟลเดอร์ `google-apps-script` มีโค้ด Apps Script สำหรับใช้เป็น Web App endpoint

ขั้นตอนโดยสรุป:

1. สร้าง Google Sheet สำหรับเก็บข้อมูล
2. เปิด Apps Script แล้วนำโค้ดจาก `google-apps-script/` ไปใช้งาน
3. Deploy เป็น Web App
4. นำ Web App URL มากำหนดในหน้าเว็บส่วน Google Sheet

ดูรายละเอียดเพิ่มเติมได้ใน `google-apps-script/README.md`

## Deploy ขึ้น GitHub Pages

1. สร้าง repository บน GitHub
2. อัปโหลดไฟล์ทั้งหมดในโฟลเดอร์นี้ขึ้น repository
3. ไปที่ `Settings > Pages`
4. เลือก branch ที่ต้องการ publish
5. ตั้งค่า root เป็นโฟลเดอร์ที่มี `index.html`
6. กด Save แล้วรอ GitHub Pages สร้าง URL

หากใช้ repository root เป็นที่เก็บไฟล์นี้โดยตรง GitHub Pages จะอ่าน `index.html` ได้ทันที

## หมายเหตุ

ข้อมูลที่ยังไม่ได้เชื่อม Google Sheet จะถูกเก็บใน browser ของเครื่องผู้ใช้ผ่าน `localStorage` หากล้างข้อมูล browser หรือเปลี่ยนเครื่อง ข้อมูลในเครื่องอาจไม่ตามไปด้วย จึงควรตั้งค่า Google Sheet sync หากต้องการใช้งานจริงร่วมกันหลายคน
